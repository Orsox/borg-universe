# Skill - Borg Change Execution

## Procedure

1. Confirm the work package prompt starts with `<nano-implant>`.
2. Confirm prepared workspace metadata matches the assigned package.
3. Read relevant code and tests before editing.
4. Implement only the approved package scope.
5. Add or update focused tests for changed behavior.
6. Update technical documentation, migrations, configuration, or templates only when required by the plan.
7. Run the listed validation commands when available.
8. Report changed files, tests, deviations, blockers, and residual risk.

## Rules

- Reuse existing project conventions and helpers.
- Do not refactor unrelated code.
- Do not silently skip tests; report missing tools or fixtures.
- Stop if plan, workspace, or dependency information conflicts.
