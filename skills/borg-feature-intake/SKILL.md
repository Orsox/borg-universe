# Skill - Borg Feature Intake

## Procedure

1. Extract the user's desired capability in one sentence.
2. Identify the beneficiary and expected value.
3. Separate in-scope behavior from non-goals.
4. Record constraints, dependencies, risks, and acceptance signals.
5. Detect missing information.
6. Ask only blocking questions; otherwise write assumptions.
7. Produce the Feature Definition.

## Output Template

```markdown
# Feature Definition

## Goal

## Value

## Scope

## Non-Goals

## Acceptance Signals

## Risks

## Dependencies

## Missing Information

## Assumptions
```

## Rules

- Keep the definition compact.
- Avoid implementation design unless required to clarify scope.
- Mark every inferred fact as an assumption.
