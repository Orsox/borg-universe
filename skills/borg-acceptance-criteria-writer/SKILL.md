# Skill — Borg Acceptance Criteria Writer

## Purpose
Transform vague task goals into verifiable acceptance criteria.

## Responsibilities
- Define concrete completion checks
- Ensure tasks are testable
- Avoid ambiguous outcomes

## Example

Instead of:
"Implement SPI layer"

Use criteria such as:
- SPI helper API exists
- Chip select handling centralized
- Functions return defined error codes
